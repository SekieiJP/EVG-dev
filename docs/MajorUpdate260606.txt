# 機能正式実装
以下の機能実装では、最適化のために過去データとの互換性を失う変更をしてもよい。
 - 個人戦績表示機能。
   - elevator-game-requirements.md の “10.5 成績指標「Skill」”で定義されたSkill値について、現在はStage Skillだけが計算されている。複数ステージをまたいだプレイヤー毎のSkillが計算され、Playerタブに随時表示され、結果発表フェーズでは個人Skillの上昇値差分が緑文字で(+120.52)のように表示されるようにして。
   - Historyタブの表示項目が elevator-game-requirements.md “10.5.4 表示する指標”で更新されているので、追従して修正。
 - 過去のゲーム・ステージ履歴表示機能。
    - firebaseのrooms/elevator-game-live/completedGames 傘下の情報を確認できるように。
    - 表示に適するようにスキーマを改修してもよい。

# 機能変更
 - 新しいゲームが開始された後、過去に参加したことのある端末でPlayer画面にアクセスすると、データを引き継いで、氏名入力不要で参加できるように。
 - Player画面の「過去のデータを引き継いではじめる」では、名前の入力を求められないように変更。
    - データベースで一致するUUIDが見つかれば続きからプレイでき、見つからない場合はエラーを返す。
 - Player画面で、Firebase移行に伴い必要なくなった「次へ」ボタンを削除。

# 機能追加: イベント種別追加
 - E7 入場料: チケット購入者全員に一律マイナスM点
   (そのステージ不参加の場合は適用されない)
 - E8 完乗ボーナス: 下車成功したらM点

# 不具合修正
 - 「締切まで」の秒数再描画により、フォームのフォーカスが外れたり、入力がリセットされたりする問題を修正。
 - 「退室」ボタンでプレイヤーを退室させた時、選択していない他プレイヤーが退室させられる問題を修正。
 - 結果発表アニメーション中にHost画面を再読み込みすると、結果発表が終わらなくなる問題を修正。
 - 仕様に反して、最終ステージであっても中間ランキングの右カラムに合計得点が表示されてしまいネタバレとなる問題を修正。
