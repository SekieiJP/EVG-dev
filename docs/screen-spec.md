# Screen Spec

## 共通

`game/index.html` は上部タブまたはクエリパラメータで表示を切り替える。

- `?view=player`
- `?view=host`
- `?view=screen`
- `?view=history`
- `?view=settings`

## プレイヤー

- 未参加時は名前入力のみを通常表示し、UUID入力は「データを引き継いではじめる」の折りたたみ内に表示する。
- 同一ブラウザで登録済みUUIDが現在ルームに存在する場合は、氏名入力画面を経由せず自動的にプレイヤー画面を表示する。
- 参加後は現在フェーズ、ステージ条件、合計得点、Skillを表示する。
- スマホ縦画面では、総階数・定員・上昇報酬・運賃の基本パラメータを1行4列で表示する。
- チケット購入受付中と締切カウントダウン中は、乗車階、降車階、最大2問の予想回答、棄権ボタンを表示する。
- 禁止階を含む投票は送信前に確認ダイアログを出す。
- 結果フェーズではステージ得点、乗車成功点、イベント補正、チケット代、実上昇、StageSkillを表示する。

## ホスト

- 初回表示でパスワード認証を行う。
- ヘッダー右側には「現在ステージ数/総ステージ数, 現在ステージ名」「現在フェーズ」「参加人数」を並べて表示する。
- 進行ボタンは現在フェーズで押せるものだけを有効にする。
- 参加者一覧には名前、UUID、入力状況、累積得点、Skillを表示する。
- 設定JSONのexport/importと、個別ステージJSONの追加importに対応する。
- 操作ログを直近100件表示する。
- スクリーン音量とミュート値をルーム状態に保存する。

## スクリーン

- 初回に「準備完了」ボタンを表示し、AudioContextを起動する。
- 参加受付中は参加URLと参加者名を表示する。
- ステージ説明・投票受付中はステージ条件と投票進捗を表示する。ステージ条件の表示名は `総階数`、`定員`、`上昇報酬`、`運賃` とする。
- チケット購入人数は円グラフで表示し、購入済み人数/参加人数を中央に表示する。
- 締切カウントダウンは大きな秒数表示にする。
- 締切カウントダウン終了後は、ギリギリで送信されたデータを受け取るため、3秒間「移動中…」を表示してから結果発表へ進む。
- 結果発表は横画面投影を主対象とし、かごは固定表示、フロア表示がスクロールしてかごが上昇しているように見える演出にする。
- 結果発表中、現在到達している階より上の階の結果は非表示にし、上昇アニメーションに合わせて順次表示する。
- 結果発表画面下部にはプレイヤー別の現在スコアを表示し、各階到達ごとに更新する。運賃は発表開始時に減算、上昇報酬は各階で加算、ボーナスは獲得が確定した階で加算する。
- 結果発表では全プレイヤーの待機、乗車、乗車中、下車、強制下車を各階に表示する。プレイヤー名は先頭5文字まで表示し、乗降時は暫定的に1秒間スクロール停止する。
- 結果発表では強制下車階と上位結果を併記する。
- 2ステージ目以降を含む中間ランキングでは、今ステージのランキングと全体を通してのランキングを左右に併記する。
- 最終結果は累積順位を大きく表示する。

## 表記

- 人の目に触れる画面では `N` は `総階数`、`X` は `定員`、`P` は `上昇報酬`、`Q` は `運賃`、`R` は `ボーナス倍率` と表記する。
- 眺望ボーナスは `眺望: 乗車成功時、降車階で得点+階数x2` のように、条件と加点を端的に表示する。
- 得点は整数、Skillは小数点以下2桁まで表示する。

## 戦歴

- 全体ランキングと選択プレイヤーの集計値を表示する。
- 個人統計12項目のうち、現在実装済みデータから算出できる値を表示する。

## 設定

- UUIDの表示・コピー・インポート。
- 名前変更。進行中ステージでは次ステージ反映として保存する。
- クライアント通信ログの閲覧。
