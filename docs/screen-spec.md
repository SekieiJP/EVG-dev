# Screen Spec

## 共通

`game/index.html` は上部タブまたはクエリパラメータで表示を切り替える。

- `?view=player`
- `?view=host`
- `?view=screen`
- `?view=history`
- `?view=settings`

## プレイヤー

- 未参加時は名前とUUID引き継ぎ入力を表示する。
- 参加後は現在フェーズ、ステージ条件、合計得点、Skillを表示する。
- チケット購入受付中と締切カウントダウン中は、乗車階、降車階、最大2問の予想回答、棄権ボタンを表示する。
- 禁止階を含む投票は送信前に確認ダイアログを出す。
- 結果フェーズではステージ得点、乗車成功点、イベント補正、チケット代、実上昇、StageSkillを表示する。

## ホスト

- 初回表示でパスワード認証を行う。
- 進行ボタンは現在フェーズで押せるものだけを有効にする。
- 参加者一覧には名前、UUID、入力状況、累積得点、Skillを表示する。
- 設定JSONのexport/importと、個別ステージJSONの追加importに対応する。
- 操作ログを直近100件表示する。
- スクリーン音量とミュート値をルーム状態に保存する。

## スクリーン

- 初回に「準備完了」ボタンを表示し、AudioContextを起動する。
- 参加受付中は参加URLと参加者名を表示する。
- ステージ説明・投票受付中はステージ条件と投票進捗を表示する。
- 締切カウントダウンは大きな秒数表示にする。
- 結果発表は横画面投影を主対象とし、上昇するエレベーターをカメラが追うように、フロア表示が下階から上階へスクロールする演出にする。
- 結果発表では強制下車階と上位結果を併記する。
- ランキング・最終結果は累積順位を大きく表示する。

## 戦歴

- 全体ランキングと選択プレイヤーの集計値を表示する。
- 個人統計12項目のうち、現在実装済みデータから算出できる値を表示する。

## 設定

- UUIDの表示・コピー・インポート。
- 名前変更。進行中ステージでは次ステージ反映として保存する。
- クライアント通信ログの閲覧。
